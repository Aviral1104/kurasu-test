Feature 1: Notes System — Per-Video Smart Notes
What it does

Every video gets its own notes panel. User can type manually OR AI can auto-generate a summary from the YouTube transcript. Notes are saved to Firestore per video_id, tagged with timestamps for easy reference.

Core feature
Uses YouTube captions API
How to build it
  • When user clicks "take notes" on a video → modal pops up
  • Textarea for manual notes + "AI Summary" button
  • Button triggers: fetch YouTube captions → send to Gemini → populate textarea
  • Save to Firestore: /users/{uid}/courses/{courseId}/videos/{videoId}/notes
Tech stack
  • Frontend: React modal component with textarea
  • API route: /api/generateNotes (Node.js)
  • YouTube captions: youtube-captions-scraper npm package (free, no API key needed)
  • AI: Gemini API (free tier, ~$0 cost at scale)
  • Database: Firestore
Database schema
/users/{uid}/courses/{courseId}/videos/{videoId}/notes { content: string // manual notes or AI summary aiGenerated: boolean // true if from Gemini transcript: string // cached YouTube transcript keyPoints: string[] // 5 bullet points from AI createdAt: timestamp updatedAt: timestamp }

Given this YouTube video transcript, create: 1. A 200-word summary 2. 5 key takeaways as bullet points 3. 3 review questions to test understanding Transcript: {transcript} Return as JSON: { "summary": "...", "keyPoints": [...], "quiz": [...] }
Feature 2: Auto-Generated AI Summary from YouTube Transcript
What's the problem?

When a user imports a YouTube playlist, they see 50 videos but have no idea what each one covers. They have to click each one to read the description. Takes forever. Solution: show a 1-sentence AI summary of each video before they even click.

Huge UX win
Implementation
  • When course is imported, loop through all videos
  • For each video: fetch caption → send first 200 words to Gemini
  • Gemini: "Summarize in 1 sentence what this video is about"
  • Cache result per video_id in Firestore (never re-call for same video)
  • Display in course list: video title + AI summary + metadata
Cost optimization

Key insight: Cache aggressively. If 100 users import the same NeetCode playlist, you only call Gemini once for each video (not 100 times). Store summaries under videos/{videoId} in a shared collection.

  • Shared cache saves 99% of API calls
  • Cost per user → near $0
  • Gemini free tier: ~1M tokens/day = 50,000+ summaries

Feature 3: Notion-Like Schedule + Learning Plan
What it is

A calendar + task board view where users can plan their learning week. "I want to finish 5 videos by Friday" — system creates daily targets, shows progress, and sends reminders. Like Notion but optimized for learning goals.

Schedule management
Week + day views
Core components
  • Goal setter: "I want to complete X course by DATE" → auto-generate daily breakdown
  • Calendar view: See what you planned for each day of the week
  • Drag-and-drop: Move videos between days, adjust schedule
  • Daily checklist: "Today: watch videos 3–5 of System Design" with checkboxes
  • Progress tracking: Visual bar showing course % done + time left
  • Email reminders: "You planned 3 videos for today. You've done 1. 2 to go."
How it works — step by step

1. User sets goal: "Complete NeetCode 250 by June 30"

2. System calculates: 250 videos ÷ 30 days = ~8 videos/day

3. Creates schedule: Day 1: videos 1–8, Day 2: 9–16, etc.

4. User can adjust: "Actually, I want to do 5/day instead"

5. Calendar regenerates: New target date calculated

6. Daily email: "Your plan for today: videos 1–5. Progress: 0/5"

7. Completion bonus: Extra XP if user finishes day on time

Database schema
/users/{uid}/schedules/{scheduleId} { courseId: string goalDueDate: timestamp videosPerDay: number createdAt: timestamp } /users/{uid}/schedules/{scheduleId}/days/{date} { date: string // "2026-05-01" videoIds: string[] // which videos to do today completed: string[] // which ones user finished targetCount: number // 8 (expected to do) }

How These 3 Features Work Together
The Flow
  • User imports NeetCode playlist
  • System fetches transcripts → AI summarizes all videos (cached)
  • User sees list: "Video 1: Two Sum. Summary: Find two numbers that add up to target."
  • User clicks "Schedule this course" → sets goal date
  • Calendar auto-fills with daily targets
  • User watches Video 1 → takes notes → marks complete
  • Calendar updates to show progress
Tech dependencies
  • All 3 features share Gemini API calls
  • All 3 store data in Firestore
  • Transcripts cached to avoid re-fetching
  • Resend for email reminders (3000/mo free)
  • Total cost at 1000 users: ~$5–10/month (mostly Firestore reads)
Why this wins
  • No competitor does this combo: Notes + summaries + scheduling together
  • Sticky product: User plans a course → comes back daily to check schedule
  • Completion engine: Schedule creates accountability, drives finishes
  • Moat: User data (notes + schedule history) makes switching hard
