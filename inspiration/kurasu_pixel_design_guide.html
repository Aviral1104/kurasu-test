How to create custom pixel icons
Free tools to draw pixel art Recommended

You don't need Photoshop. These tools are free, browser-based, and export clean PNGs or SVGs.

  • Piskel (piskelapp.com) — free, browser-based, exports PNG sprite sheets + animated GIFs
  • Lospec Pixel Editor (lospec.com) — cleanest UI, palette tools built in
  • Aseprite — paid ($20, one-time) but the industry standard if you get serious
  • LibreSprite — free fork of Aseprite, same features
Icon sizes to use Spec

Pixel art must be drawn small then scaled up with image-rendering: pixelated — never upscale with smoothing.

  • Nav icons: draw at 16×16, display at 32×32
  • Course card icons: draw at 16×16, display at 48×48
  • Rank badges: draw at 8×8, display at 24×24
  • Avatar frames: draw at 32×32, display at 64×64
  • Animated sprites: 16×16 per frame, 4–8 frames
/* Critical CSS — never forget this */ img.pixel, canvas.pixel { image-rendering: pixelated; image-rendering: crisp-edges; }
Which icons to make first Priority list

Don't draw everything at once. Start with these 8 — they cover 90% of what users see.

  • ⚔️ Sword — for course/dungeon cards
  • 💎 Crystal — for XP / premium
  • 🔥 Flame — for streak
  • 📜 Scroll — for notes / summaries
  • ⚡ Lightning — for quiz / fast action
  • 🛡️ Shield — for flashcards / defense
  • 🌀 Gate portal — for course import
  • 💀 Skull — for failed quiz / penalty

Pixel animations — what to build and how
Sprite sheet animation (CSS) No library needed

Draw a flame or portal in Piskel as 4–8 frames. Export as a horizontal sprite sheet PNG. Animate with CSS steps().

/* 4-frame flame: 64px wide sheet, 16px per frame */ .flame-sprite { width: 16px; height: 16px; background: url('/sprites/flame.png'); image-rendering: pixelated; animation: burn 0.4s steps(4) infinite; } @keyframes burn { from { background-position: 0 0; } to { background-position: -64px 0; } }
Rank-up particle burst CSS only

On course completion, spawn 12 small pixel squares that explode outward. Pure CSS, no canvas needed.

/* Pixel particle */ .particle { width: 6px; height: 6px; background: #a78bfa; position: absolute; image-rendering: pixelated; animation: burst 0.6s ease-out forwards; } @keyframes burst { 0% { transform: translate(0,0) scale(1); opacity:1; } 100% { transform: translate(var(--dx), var(--dy)) scale(0); opacity:0; } } /* Set --dx and --dy randomly per particle in JS */
Typewriter system message JS

The System speaks to the user on key events — quest complete, rank up, streak milestone. Typed out character by character with a blinking cursor.

function systemSpeak(msg, el) { el.textContent = ''; let i = 0; const type = () => { if (i < msg.length) { el.textContent +=msg[i++]; setTimeout(type, 40); } }; type(); } // Usage: systemSpeak( "You have cleared the dungeon, Hunter." , document.querySelector('.system-msg') );
Pixel screen flash on XP gain Micro-interaction

Every time XP is earned, the XP counter flashes and a +XP bubble pops up and floats away. Feels like an RPG damage number.

/* Floating XP number */ .xp-pop { position: absolute; font-family: monospace; font-size: 13px; color: #a78bfa; font-weight: 700; pointer-events: none; animation: floatUp 1s ease-out forwards; } @keyframes floatUp { 0% { opacity: 1; transform: translateY(0); } 100% { opacity: 0; transform: translateY(-40px); } }
Firebase — what to store where
Firestore collections Structure
  • users/{uid} — rank, xp, streak, avatar, theme, createdAt
  • users/{uid}/courses — title, sourceType, sourceUrl, rank, progress
  • users/{uid}/quests — daily quests, completion status, resetAt
  • users/{uid}/shadows — flashcard collection per video
  • videos/{videoId} — shared AI notes cache (saves API calls)
Firebase Storage — pixel assets Free tier: 5GB
  • Store sprite sheets at /sprites/*.png
  • Store avatar uploads at /avatars/{uid}.png
  • Store rank-up sounds (optional) at /sfx/*.mp3
  • Cache AI-generated notes as JSON at /notes/{videoId}.json — avoid re-calling Gemini for same video
